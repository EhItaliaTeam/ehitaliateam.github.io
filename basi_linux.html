1. La Shell - Bash

La shell o il terminale è uno strumento davvero utile. Bash è la shell standard sulla maggior parte delle distribuzioni Linux.

Vavigazione

cd - Cambia directory

cd ~ - Tornare alla home directory

cd - - Tornare alla directory precedente

Guardare/leggere file

ls -ltr - Ordina l'elenco per ultima modifica. -time -reverse

file - Mostra le informazioni sul file. Che tipo di file è. Ad esempio, se si tratta di un file binario o di testo

less - Restituisce File di output ma solo visualizzandolo una pagina per volta, permettendo di scorre in avanti e all'indietro e di effettuare ricerche.

Use /searchterm per cercare. È lo stesso comando di im, n per scorrere fino al risultato di ricerca successivo. Premere q per uscire.

Trovare file

Esistono principalmente tre modi per trovare file su Linux: find, locate, and which.

Find

Find è più lento di locate ma molto più approfondito. Puoi cercare i file in modo ricorsivo e usare regex e molte altre funzionalità.

# Questo invierà tutte le autorizzazioni negate agli output su dev / null.
find / -name file 2>/dev/null

Locate

Locate è molto veloce perché si basa su un database interno. Quindi, per averlo aggiornato è necessario eseguire: 

sudo updatedb

Quindi puoi facilmente trovare cose come questa:

locate filename

Which

Emette il percorso del binario che stai cercando. Cerca tra le directory definite nella variabile $PATH. 

which bash
# Outputs: /bin/bash

2. Modifica del testo

Stdin

Stdin sono i dati che vengono inseriti nel programma. Un esempio di un programma che richiede dati stdin è cp . Affinché il programma possa fare qualsiasi cosa, ha bisogno di dati di input. Ad esempio cp file1 copy_of_file1 . Qui file1 e copy_of_file1 è lo stdin.

Quindi lo Stdin predefinito proviene dal file STDIN che è una rappresentazione di file di testo dell'input da tastiera. Ma spesso non vogliamo inserire elementi dalla tastiera, a volte vogliamo inserire qualcosa in un programma che proviene da un altro file. Questo è quando possiamo usare il simbolo di reindirizzamento: & gt; .

Un altro esempio potrebbe essere cat < my_text_file.txt.I dati da my_text_file.txt verranno usati come input invece della tastiera.

Il descrittore di file per stdin è: 0

Stdout

Stdout sono i dati che vengono estrapolati dal programma. Per esempio, quando usiamo il comando cat file1 quel dato/text che viene emesso è lo stdout. Lo stesso con il programma ls. Non tutti i programmi hanno stdout. Per esempio quando usiamo mv or cp non si ottiene alcun ritorno dal programma.

Il stdout puo' essere reindirizzato su file usando il simbolo > and >>. Quindi possiamo digitare:

ls > result_of_ls.txt
# ora il risultato viene scritto sul file result_of_ls.txt
ls >> result_of_ls.txt
# Questo scrive in modalita' append il dato sul file result_of_ls.txt

Un'altra caratteristica incredibilmente utile è la pipe caratteristica, rappresentata da questo simbolo |. Prende lo stdout e lo reindirizza in un altro programma. Ecco un esempio:

ls -la | less

Questo prenderà lo stdout da ls -la e inoltrarlo / reindirizzarlo nel file less programma. Usando il pipe ora si possono concatenare diversi comandi.

Il descrittore di file per stdout is: 1

Stderr

Stderr è il flusso utilizzato per inviare messaggi di errore. Quindi se un programma fallisce per qualsiasi motivo. Ad esempio, se proviamo a copiare un file che non esiste, questo sarà l'output stdrr:

cp thisfiledoesnotexist aaaaaaaaaa
cp: cannot stat 'thisfiledoesnotexist': No such file or directory

Questo è un modo comune per stderr di presentarsi, appena uscito direttamente nel terminale. Ma a volte stderr viene inviato a un file di registro.

Stderr è utile perché con esso possiamo separare tra stdout e stderr. Tuttavia, per gli occhi potrebbe essere difficile distinguere quale sia l'output stdout e qual è l'output stderr.

Un modo semplice per determinare è l'output stderr o stdoutè semplicemente reindirizzarlo in un file. Perché di default reindirizzi solo stdout, e non stderr.

cp thisfiledoesnotexist aaaaaaaaaa > result.txt
cp: cannot stat 'thisfiledoesnotexist': No such file or directory
# Se ora guardiamo result.txt scopriremo che è vuoto. Poiché il testo dell'errore che abbiamo ricevuto non può essere reindirizzato nel file di testo, poiché è stderr e non stdout.

cut

Questo è un comando utile per tagliare del testo.

Abbiamo il seguente testo e vogliamo tagliare l'indirizzo IP.

64 bytes from 192.168.0.1: icmp_req=1 ttl=255 time=4.86 ms

cut -d" " -f4

-d sta per il delimitatore e -f per il campo

tr - Tradurre

Trasforma tutte le lettere in maiuscolo.

tr "[:lower:]" "[:upper:]" < file1 > file2

Example
Remove character

# Rimuove caratteri
cat file.txt | tr -d "."
# rimuove e rimpiazza
# Rimuovi tutti i punti e sostituiscili con un trattino basso.
cat file.txt | tr "." "_"

http://www.thegeekstuff.com/2012/12/linux-tr-command/

awk

Awk è uno strumento avanzato per la modifica di file di testo.Il suo linguaggio di programmazione può diventare piuttosto complesso. Awk scorre l'intero file riga per riga.

Questa è la struttura di base di un comando awk.

awk '/search_pattern/ { action_to_take_on_matches; another_action; }' file_to_parse

The search pattern takes regex.
You can exclude the search portion or the action portion.

Questo stampa solo ogni riga del file.

awk '{print}' filename

Filtering out specific ip-address:

awk '/172.16.40.10.81/' error.log

Ora stampiamo la quarta colonna di quel file, possiamo semplicemente eseguire il pipe di questo per tagliare, ma possiamo anche usare awk per questo, in questo modo:

awk '/172.16.40.10.81/ {print $4}' error.log
# Un altro esempio
awk '{print $2,$5;}' error.txt
Questo printa colonna 2 e 5.

Possiamo usare il flag -F per aggiungere un delimitatore personalizzato.

awk -F ':' '{print $1}' test.txt

Quindi, se stai manipolando del testo, potresti voler avviare l'output con alcune informazioni sulle colonne o qualcosa del genere. Per fare ciò possiamo usare la parola chiave BEGIN.

awk 'BEGIN {printf "IP-address \tPort\n"} /nop/ {print $3}' test.txt | head
awk 'BEGIN{printf "IP-address \tPort\n"} /nop/ {print $3} END {printf "End of the file\n"}' test.txt | tail

Here we are printing IP-address PORT to the first line of the file.

3. Gestione utenti

Per aggiungere un utente:

adduser NameOfUser
# In alcune macchine
useradd nameOfUser

Per aggiungere l'utente al gruppo sudo:

adduser NameOfUser sudo

Su alcune macchine potremmo non essere in grado di modificare il file sudoers perché non abbiamo una shell interattiva,in questo caso puoi semplicemente reindirizzare il testo nel file, in questo modo:

echo "username ALL=(ALL) ALL" >> /etc/sudoers

Controlla quali utenti sono nel gruppo sudo:

cat /etc/group | grep sudo

Cambia utente nel terminale:

su NameOfUser

Rimuovi/elimina user:

sudo userdel NameOfUser

4. Permessi

ls -la

Mostra tutti i file e le directory e le relative impostazioni di autorizzazione.

drwxrwxrwt 2 root root 4,0K ago  3 17:33 myfile

Qui abbiamo 10 lettere all'inizio. Il primo d mostra che si tratta di una directory.
Le prossime tre lettere sono per la lettura, w per scrivere x ed eseguire. I primi tre appartengono al proprietario, i secondi tre al gruppo e gli ultimi tre a tutti gli utenti.

https://linuxjourney.com/lesson/file-permissions

5. Processi

Per visualizzare informazioni relative ai processi di sistema è possibile utilizzare ps . 

ps -aux

-a stands for all -u sta per tutti i processi di tutti gli utenti -x sta per tutti i processi che non eseguono a tty

Se esegui questo comando probabilmente vedrai un risultato piuttosto grande. Nella colonna per command vedrai quale comando è stato eseguito.Ogni processo ha un numero di identificazione del processo (PID). Qualcosa che vedrai anche nell'output.

Tutti questi processi possono essere effettivamente trovati in /proc.Usare /proc/[pid]. Su /proc puoi trovare informazioni sul sistema e puoi effettivamente cambiare il sistema se cambi quei file! But more on that later. Quello che volevo spiegare è che se guardiamo l'output da ps vediamo che alcuni comandi sono tra parentesi. Come questo:

root        10  0.0  0.0      0     0 ?        S    ene14   0:00 [watchdog/0]
root        11  0.0  0.0      0     0 ?        S    ene14   0:00 [watchdog/1]
root        12  0.0  0.0      0     0 ?        S    ene14   0:00 [migration/1]
root        13  0.0  0.0      0     0 ?        S    ene14   0:00 [ksoftirqd/1]

Questi sono generalmente processi del kernel e puoi tranquillamente supporre che nessun utente li abbia avviati.

Se vuoi monitorare i processi in tempo reale puoi usarli top o htop. top viene preinstallato sulla maggior parte delle distribuzioni. Ma htop è davvero molto più bello.

Per htop the F1-10 le chiavi potrebbero attivare eventi del sistema operativo. Quindi puoi usare le scorciatoie invece.

6. Packages

Something that difference Linux from windows is how it handles installing new software. In windows you usually have to google around and then click on random scary download buttons that might fuck up your computer, or not. It's like a constant lottery where you win by no installing malware. In Linux that is usually not really an issue. That is because distros have their own software repositories from where you can download your software. This is kind of like an app-store except everything is free.

The different major branches of teh GNU/Linux OS have their own software repositories. Ubuntu has their own, debian has their own, and so on.

Different distros also have their own package-amangers. For example, Debian and ubuntu uses apt, while Redhat uses rpm, and Arch uses pacman. You should strick to your own package-manager, because even though chaning package-manager is possible it will probably just cause you more headache than benefits.

Install package

Example of how to install something with apt:

sudo apt-get install nmap

If you only have a .deb file you do this to install from the terminal:

sudo dpkg -i /path/to/deb/file

sudo apt-get install -f
